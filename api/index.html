API Documentation

Task Management

POST /api/tasks

Create a new task.

Example request:

POST /api/tasks HTTP/1.1
Content-Type: application/json

{
    "type": "content_generation",
    "parameters": {
        "keyword": "example keyword",
        "intent": "informational"
    },
    "priority": "high"
}

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

{
    "success": true,
    "task_id": "123456",
    "status": "queued"
}

Agent Communication

GET /api/communication/flows

Get inter-agent communication flows.

Query Parameters:

  • start_time – Start time for filtering (ISO format)

  • end_time – End time for filtering (ISO format)

Status Codes:

  • 200 OK – No error

GET /api/communication/realtime

Stream real-time agent interaction data.

Status Codes:

  • 200 OK – Server-sent events stream started

GET /api/communication/analytics

Get communication patterns analytics.

Query Parameters:

  • timeframe – Time frame for analysis (e.g., ‘24h’)

Status Codes:

  • 200 OK – No error

Process Timeline

GET /api/timeline/process

Get detailed process stage tracking.

Query Parameters:

  • process_id – ID of the process to track

  • include_details – Include additional details (boolean)

Status Codes:

  • 200 OK – No error

GET /api/timeline/decisions

Get decision point logging.

Query Parameters:

  • process_id – ID of the process

  • start_time – Start time for filtering

  • end_time – End time for filtering

Status Codes:

  • 200 OK – No error

Resource Monitoring

GET /api/resources/allocation

Get detailed resource allocation tracking.

Query Parameters:

  • resource_type – Type of resource to monitor

  • timeframe – Time frame for analysis

Status Codes:

  • 200 OK – No error

GET /api/resources/usage-patterns

Get resource usage patterns.

Query Parameters:

  • resource_type – Type of resource

  • start_time – Start time for analysis

  • end_time – End time for analysis

Status Codes:

  • 200 OK – No error

GET /api/resources/bottlenecks

Get resource bottleneck identification.

Query Parameters:

  • analysis_period – Period for analysis (default: ‘24h’)

  • threshold – Bottleneck threshold (default: 0.8)

Status Codes:

  • 200 OK – No error

System Status

GET /api/system/status

Get overall system status.

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

{
    "agents": {
        "content": {"status": "healthy"},
        "seo": {"status": "healthy"},
        "keyword": {"status": "healthy"},
        "crawler": {"status": "healthy"}
    },
    "tasks": 10,
    "active_tasks": 2,
    "system_health": "optimal",
    "last_update": "2024-02-04T12:00:00Z"
}

Error Responses

ANY /api/*

Common error responses for all endpoints.

Example error response:

HTTP/1.1 400 Bad Request
Content-Type: application/json

{
    "error": "Invalid parameters",
    "details": "Missing required field: type"
}
Status Codes:

  • 400 Bad Request – Bad request

  • 401 Unauthorized – Authentication required

  • 403 Forbidden – Forbidden

  • 404 Not Found – Resource not found

  • 429 Too Many Requests – Too many requests

  • 500 Internal Server Error – Server error
